🧩 1.1 类与对象

类是 C++ 面向对象编程的核心，它将数据（成员变量）和行为（成员函数）封装在一起。 对象是类的实例。通过 class 或 struct 关键字定义类， 使用 public / private / protected 控制访问权限。

📝 关键概念

  • class 默认访问权限是 private；struct 默认是 public
  • 成员函数可以在类内声明、类外定义（使用 类名::函数名）
  • this 指针：指向调用该成员函数的对象本身
  • const 成员函数：承诺不修改成员变量，声明为 返回类型 函数名() const

💻 示例代码

class Student {
private:
    string name;
    int score;
public:
    Student(string n, int s) : name(n), score(s) {}
    void print() const {
        cout << name << ": " << score << endl;
    }
    int getScore() const { return score; }
};

int main() {
    Student s("小明", 95);
    s.print();  // 输出: 小明: 95
}

🏗️ 1.2 构造函数与析构函数

构造函数在对象创建时自动调用，负责初始化；析构函数在对象销毁时自动调用，负责清理资源。

📝 关键概念

  • 默认构造函数：无参构造函数。若未定义任何构造函数，编译器自动生成
  • 初始化列表：成员(值), 成员(值) — 效率高于在函数体内赋值，const 和引用成员必须用初始化列表
  • 委托构造函数：C++11，一个构造函数调用另一个
  • 析构函数：~类名()，无参数无返回值，用于释放资源（如 delete 堆内存、关闭文件）
  • RAII：资源获取即初始化，利用构造/析构自动管理资源

💻 示例代码

class DynamicArray {
private:
    int* data;
    size_t size;
public:
    // 构造函数：分配内存
    DynamicArray(size_t n) : size(n), data(new int[n]{}) {
        cout << "构造: 分配 " << n << " 个 int" << endl;
    }
    // 析构函数：释放内存
    ~DynamicArray() {
        delete[] data;
        cout << "析构: 释放内存" << endl;
    }
};

🧬 1.3 继承

继承允许派生类复用基类的成员。C++ 支持 public、protected、private 三种继承方式，以及多继承。

📝 关键概念

  • public 继承：基类的 public → 派生类的 public，protected → protected
  • protected 继承：基类的 public/protected → 派生类的 protected
  • private 继承：基类的 public/protected → 派生类的 private（默认）
  • 多继承：class D : public A, public B
  • 菱形继承问题：用 virtual 虚继承解决
  • 派生类构造函数必须调用基类构造函数（否则调用默认）

💻 示例代码：多继承与虚继承

class Animal {
protected:
    string name;
public:
    Animal(string n) : name(n) {}
    virtual void speak() { cout << name << " 叫了"; }
};
class Dog : public Animal {
public:
    Dog(string n) : Animal(n) {}
    void speak() override { cout << name << ": 汪汪!"; }
};

🎭 1.4 多态与虚函数

多态是 OOP 的核心特性：通过基类指针/引用调用派生类重写的函数。虚函数 + 基类指针 是运行时多态的基础。

📝 关键概念

  • 虚函数：用 virtual 关键字声明，通过虚函数表（vtable）实现动态绑定
  • override：C++11 关键字，显式声明重写，防止拼写错误
  • final：禁止进一步重写，或禁止类被继承
  • 纯虚函数：virtual void f() = 0; — 使类成为抽象类，不能实例化
  • 虚析构函数：基类析构函数应为 virtual，确保 delete 基类指针时正确调用派生类析构

💻 示例代码：多态经典案例

class Shape {
public:
    virtual double area() const = 0;  // 纯虚函数
    virtual ~Shape() {}              // 虚析构
};
class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    double area() const override { return 3.14 * r * r; }
};
void printArea(const Shape& s) {
    cout << s.area() << endl;  // 多态调用
}

➕ 1.5 运算符重载

C++ 允许重载大部分运算符，使自定义类型可以像内置类型一样自然地使用运算符。

📝 关键概念

  • 可重载为成员函数（左操作数是 *this）或友元/全局函数（双操作数对称处理）
  • ++/-- 区分前后缀：后缀有 int 哑元参数
  • operator<< / operator>> 必须为友元/全局（因为左操作数是 ostream）
  • operator=、operator[]、operator()、operator-> 必须是成员函数
  • 不可重载的运算符：::、.、.*、?:、sizeof、typeid

💻 示例代码：复数类运算符重载

class Complex {
    double real, imag;
public:
    Complex(double r=0, double i=0) : real(r), imag(i) {}
    // 成员函数重载 +
    Complex operator+(const Complex& o) const {
        return Complex(real+o.real, imag+o.imag);
    }
    // 友元重载 << 输出
    friend ostream& operator<<(ostream& os, const Complex& c) {
        os << c.real << "+" << c.imag << "i";
        return os;
    }
};

🤝 1.6 友元函数与友元类

友元可以访问类的 private/protected 成员。用 friend 关键字声明函数或类为友元。

📝 关键概念

  • 友元函数：全局函数或另一个类的成员函数，声明为 friend 返回类型 函数名(参数);
  • 友元类：friend class 类名; — 该类的所有成员函数都可访问
  • 友元关系不传递、不继承（你是我的朋友，不代表你的朋友也是我的朋友）
  • 常用于运算符重载（如 operator<<）和需要访问多个类私有成员的场景

或者提供一个 getter 公有方法，但那样就不需要友元了——友元的本质就是在不暴露公有接口的情况下授权访问。

📊 1.7 静态成员

静态成员属于类本身而非某个对象，所有对象共享同一份数据。

📝 关键概念

  • 静态成员变量：类内声明，类外定义（C++17 可用 inline static 类内定义）
  • 静态成员函数：没有 this 指针，只能访问静态成员
  • 通过 类名::成员 访问，也可通过对象访问
  • 典型用途：对象计数、共享配置、工厂方法

💻 示例代码

class Counter {
    static int count;  // 声明
public:
    Counter()  { ++count; }
    ~Counter() { --count; }
    static int getCount() { return count; }  // 静态函数
};
int Counter::count = 0;  // 类外定义（必须）

📋 1.8 拷贝控制（三/五法则）

拷贝控制包括：拷贝构造、拷贝赋值、移动构造、移动赋值、析构函数。三/五法则：如果定义了其中一个，通常需要定义全部。

📝 关键概念

  • 拷贝构造函数：T(const T& other) — 深拷贝 vs 浅拷贝
  • 拷贝赋值运算符：T& operator=(const T& other) — 注意自赋值检测
  • 移动构造函数：T(T&& other) noexcept — 窃取资源
  • 移动赋值运算符：T& operator=(T&& other) noexcept
  • 三五法则：需要自定义析构 → 通常也需要自定义拷贝/移动
  • Rule of Zero：尽量让编译器生成，用 unique_ptr 等 RAII 管理资源

💻 示例代码：手搓简易 string

class MyString {
    char* data;
    size_t len;
public:
    MyString(const char* s="") : len(strlen(s)), data(new char[len+1]) {
        strcpy(data, s);
    }
    // 拷贝构造（深拷贝）
    MyString(const MyString& o) : len(o.len), data(new char[len+1]) {
        strcpy(data, o.data);
    }
    // 拷贝赋值（注意自赋值 & 释放旧资源）
    MyString& operator=(const MyString& o) {
        if (this != &o) { delete[] data; len=o.len; data=new char[len+1]; strcpy(data,o.data); }
        return *this;
    }
    ~MyString() { delete[] data; }
};
✏️ 练习题 6：移动语义

为上述 MyString 添加移动构造函数和移动赋值运算符。解释为什么它们应该标记为 noexcept。

👇 点击查看参考解答 
// 移动构造：直接窃取资源
MyString(MyString&& o) noexcept : data(o.data), len(o.len) {
    o.data = nullptr; o.len = 0;
}
// 移动赋值
MyString& operator=(MyString&& o) noexcept {
    if (this != &o) { delete[] data; data=o.data; len=o.len; o.data=nullptr; o.len=0; }
    return *this;
}
noexcept 原因：std::vector 等容器在扩容时，如果移动构造是 noexcept，会用移动而非拷贝来转移元素，提高性能。若移动可能抛异常，容器会退化为拷贝以保证安全。

📐 2.1 vector — 动态数组

STL 手搓 vector 的核心是动态扩容的连续数组：三指针（_start, _finish, _endOfStorage）+ 二倍扩容策略。

📝 关键设计

  • 三指针结构：_start（首元素）、_finish（尾后）、_endOfStorage（容量尾后）
  • size() = _finish - _start，capacity() = _endOfStorage - _start
  • push_back 时若 _finish == _endOfStorage 则扩容（通常 2 倍）
  • 扩容步骤：分配新内存 → 拷贝/移动旧元素 → 释放旧内存 → 更新三指针
  • insert 在中间插入需要移动后续元素（O(n)）
  • erase 删除元素需要前移后续元素（O(n)）

💻 最小实现框架

template<typename T>
class MyVector {
    T* _start;       // 指向数组首元素
    T* _finish;      // 指向最后一个元素的下一个位置
    T* _endOfStorage;// 指向分配内存的末尾
public:
    MyVector() : _start(nullptr), _finish(nullptr), _endOfStorage(nullptr) {}
    size_t size() const { return _finish - _start; }
    size_t capacity() const { return _endOfStorage - _start; }
    T& operator[](size_t i) { return _start[i]; }

    void push_back(const T& val) {
        if (_finish == _endOfStorage) {
            size_t oldCap = capacity();
            size_t newCap = oldCap == 0 ? 1 : oldCap * 2;
            reserve(newCap);
        }
        new(_finish) T(val);  // placement new
        ++_finish;
    }

    void reserve(size_t newCap) {
        if (newCap <= capacity()) return;
        T* newData = static_cast<T*>(operator new(newCap * sizeof(T)));
        size_t oldSize = size();
        for (size_t i = 0; i < oldSize; ++i)
            new(newData + i) T(std::move(_start[i]));  // 移动构造
        for (size_t i = 0; i < oldSize; ++i)
            _start[i].~T();  // 析构旧元素
        operator delete(_start);
        _start = newData;
        _finish = _start + oldSize;
        _endOfStorage = _start + newCap;
    }

    ~MyVector() {
        for (size_t i = 0; i < size(); ++i) _start[i].~T();
        operator delete(_start);
    }
};
✏️ 练习题 7：实现 pop_back 和 insert

1. 实现 pop_back()（移除最后一个元素）
2. 实现 insert(size_t pos, const T& val)（在 pos 位置插入 val，后续元素后移）
3. 扩容因子为什么通常选 2 而不是 3 或 1.5？

👇 点击查看参考解答 
void pop_back() {
    if (size() > 0) { --_finish; _finish->~T(); }
}

void insert(size_t pos, const T& val) {
    if (_finish == _endOfStorage) {
        size_t newCap = capacity()==0 ? 1 : capacity()*2;
        reserve(newCap);
    }
    for (T* p = _finish; p > _start+pos; --p)
        new(p) T(std::move(*(p-1)));  // 后移
    *(_start+pos) = val;
    ++_finish;
}
扩容因子：选 2 是因为均摊 O(1)；选太大浪费内存，选太小（如 1.1）频繁扩容降低性能。实际很多实现用 1.5 或 2。

🔗 2.2 list — 双向链表

STL 手搓 list 是双向循环链表，每个节点有 prev 和 next 指针。插入删除 O(1)，不支持随机访问。

📝 关键设计

  • 采用带头节点的双向循环链表：哨兵节点简化边界处理
  • 节点结构：{T data; Node* prev; Node* next;}
  • push_back：在哨兵前插入新节点
  • push_front：在哨兵后插入新节点
  • 迭代器：需要支持 ++、--、*、->
  • 任何位置的插入/删除都不会使其他迭代器失效（和 vector 不同）

💻 最小实现框架

template<typename T>
class MyList {
    struct Node {
        T data;
        Node* prev;
        Node* next;
        Node(const T& d, Node* p=nullptr, Node* n=nullptr)
            : data(d), prev(p), next(n) {}
    };
    Node* head;  // 哨兵节点
    size_t sz;
public:
    MyList() : head(new Node(T{})), sz(0) {
        head->prev = head->next = head;
    }
    void push_back(const T& val) {
        Node* n = new Node(val, head->prev, head);
        head->prev->next = n;
        head->prev = n;
        ++sz;
    }
    void push_front(const T& val) {
        Node* n = new Node(val, head, head->next);
        head->next->prev = n;
        head->next = n;
        ++sz;
    }
    size_t size() const { return sz; }
    bool empty() const { return sz == 0; }
};
✏️ 练习题 8：实现 pop_front 和迭代器

iterator begin() { return iterator(head->next); }

📚 2.3 stack — 栈

STL 手搓 栈是 LIFO（后进先出）的容器适配器，底层默认用 deque，也可用 vector 或 list。

💻 基于链表的栈实现

template<typename T>
class MyStack {
    struct Node { T data; Node* next; Node(const T& d, Node* n=nullptr) : data(d), next(n) {} };
    Node* topNode;
    size_t sz;
public:
    MyStack() : topNode(nullptr), sz(0) {}
    void push(const T& val) {
        topNode = new Node(val, topNode); ++sz;
    }
    void pop() {
        if (empty()) return;
        Node* old = topNode;
        topNode = topNode->next;
        delete old; --sz;
    }
    T& top() { return topNode->data; }
    bool empty() const { return sz == 0; }
    size_t size() const { return sz; }
};
✏️ 练习题 9：用栈实现括号匹配

编写函数 bool isValid(string s)，判断字符串中的括号 ()[] {} 是否匹配。 要求使用你自己写的 MyStack。

👇 点击查看参考解答 
bool isValid(string s) {
    MyStack<char> st;
    for (char c : s) {
        if (c == '(' || c == '[' || c == '{')
            st.push(c);
        else {
            if (st.empty()) return false;
            char t = st.top(); st.pop();
            if ((c==')'&&t!='(') || (c==']'&&t!='[') || (c=='}'&&t!='{'))
                return false;
        }
    }
    return st.empty();
}

🚶 2.4 queue — 队列

STL 手搓 队列是 FIFO（先进先出）的容器适配器。可以用链表或循环数组实现。

💻 基于循环数组的队列（环形缓冲区）

template<typename T>
class MyQueue {
    T* data;
    size_t cap;
    size_t frontIdx, rearIdx, count;
public:
    MyQueue(size_t c=16) : cap(c), frontIdx(0), rearIdx(0), count(0) {
        data = new T[cap];
    }
    void push(const T& val) {
        if (count == cap) { /* 扩容逻辑 */ }
        data[rearIdx] = val;
        rearIdx = (rearIdx + 1) % cap;
        ++count;
    }
    void pop() {
        if (empty()) return;
        frontIdx = (frontIdx + 1) % cap;
        --count;
    }
    T& front() { return data[frontIdx]; }
    bool empty() const { return count == 0; }
    size_t size() const { return count; }
    ~MyQueue() { delete[] data; }
};

⛰️ 2.5 priority_queue — 二叉堆

STL 手搓 较难 优先队列底层是二叉堆（通常是最大堆），用数组存储完全二叉树。

📝 关键设计

  • 用数组存完全二叉树：节点 i 的左子 = 2i+1，右子 = 2i+2，父 = (i-1)/2
  • 上浮（sift up / percolate up）：push 时新元素放末尾，不断与父节点比较交换
  • 下沉（sift down / heapify）：pop 时将末尾元素移到堆顶，不断与较大子节点交换
  • 默认最大堆（less），可用 greater 改为最小堆

💻 最小实现框架

template<typename T>
class MyPriorityQueue {
    MyVector<T> heap;  // 用之前写的 vector

    void siftUp(size_t idx) {
        while (idx > 0) {
            size_t parent = (idx - 1) / 2;
            if (heap[parent] >= heap[idx]) break;  // 最大堆
            std::swap(heap[parent], heap[idx]);
            idx = parent;
        }
    }

    void siftDown(size_t idx) {
        size_t n = heap.size();
        while (true) {
            size_t largest = idx;
            size_t left  = 2 * idx + 1;
            size_t right = 2 * idx + 2;
            if (left  < n && heap[left]  > heap[largest]) largest = left;
            if (right < n && heap[right] > heap[largest]) largest = right;
            if (largest == idx) break;
            std::swap(heap[idx], heap[largest]);
            idx = largest;
        }
    }
public:
    void push(const T& val) {
        heap.push_back(val);
        siftUp(heap.size() - 1);
    }
    void pop() {
        if (empty()) return;
        heap[0] = heap[heap.size() - 1];
        heap.pop_back();
        siftDown(0);
    }
    T& top() { return heap[0]; }
    bool empty() const { return heap.size() == 0; }
};

给定一个整数数组，找出第 K 大的元素。请说明应该如何设置堆（最大堆还是最小堆）以及时间复杂度。

🗂️ 2.6 unordered_map — 哈希表

STL 手搓 较难 哈希表通过哈希函数将键映射到桶（bucket），用拉链法或开放定址法解决冲突。平均 O(1) 查找。

📝 关键设计（拉链法）

  • 数组 + 链表：每个桶是一个链表头指针
  • 哈希函数：hash(key) % bucket_count
  • 负载因子：size / bucket_count，超过阈值（如 0.75）触发 rehash
  • rehash：分配更大的桶数组，将所有元素重新哈希插入
  • 节点结构：{Key k; Value v; Node* next;}

💻 最小实现框架

template<typename K, typename V>
class MyUnorderedMap {
    struct Node {
        K key; V value; Node* next;
        Node(const K& k, const V& v, Node* n=nullptr)
            : key(k), value(v), next(n) {}
    };
    Node** buckets;
    size_t bucketCount;
    size_t elemCount;
    static constexpr double LOAD_FACTOR = 0.75;

    size_t hash(const K& key) const {
        return std::hash<K>{}(key) % bucketCount;
    }

    void rehash(size_t newCap) {
        Node** oldBuckets = buckets;
        size_t oldCount = bucketCount;
        buckets = new Node*[newCap]();
        bucketCount = newCap;
        for (size_t i = 0; i < oldCount; ++i) {
            Node* cur = oldBuckets[i];
            while (cur) {
                Node* next = cur->next;
                size_t idx = hash(cur->key);
                cur->next = buckets[idx];
                buckets[idx] = cur;
                cur = next;
            }
        }
        delete[] oldBuckets;
    }
public:
    MyUnorderedMap(size_t cap=16) : bucketCount(cap), elemCount(0) {
        buckets = new Node*[cap]();
    }
    void insert(const K& key, const V& value) {
        if ((double)elemCount / bucketCount > LOAD_FACTOR)
            rehash(bucketCount * 2);
        size_t idx = hash(key);
        Node* cur = buckets[idx];
        while (cur) {
            if (cur->key == key) { cur->value = value; return; }
            cur = cur->next;
        }
        buckets[idx] = new Node(key, value, buckets[idx]);
        ++elemCount;
    }
    V* find(const K& key) {
        Node* cur = buckets[hash(key)];
        while (cur) {
            if (cur->key == key) return &cur->value;
            cur = cur->next;
        }
        return nullptr;
    }
};
✏️ 练习题 11：实现 erase 方法

为上述 MyUnorderedMap 实现 bool erase(const K& key) 方法，删除指定键的元素，成功返回 true。

👇 点击查看参考解答 
bool erase(const K& key) {
    size_t idx = hash(key);
    Node* cur = buckets[idx], *prev = nullptr;
    while (cur) {
        if (cur->key == key) {
            if (prev) prev->next = cur->next;
            else     buckets[idx] = cur->next;
            delete cur;
            --elemCount;
            return true;
        }
        prev = cur;
        cur = cur->next;
    }
    return false;
}

class MySharedPtr {

        return *this;

};
题 2：手搓简单的 BST（二叉搜索树）插入和查找（5分）

实现 BST 类，包含 insert(int val) 和 bool find(int val) 方法。

👇 参考答案 
class BST {
    struct Node { int val; Node* left; Node* right; Node(int v):val(v),left(nullptr),right(nullptr){}};
    Node* root;
public:
    BST() : root(nullptr) {}
    void insert(int val) {
        Node** p = &root;
        while (*p) {
            if (val < (*p)->val) p = &(*p)->left;
            else p = &(*p)->right;
        }
        *p = new Node(val);
    }
    bool find(int val) {
        Node* cur = root;
        while (cur) {
            if (val == cur->val) return true;
            cur = val < cur->val ? cur->left : cur->right;
        }
        return false;
    }
};

### 4.3 数据库建表 SQL

```sql
-- 访客记录表
CREATE TABLE IF NOT EXISTS visits (
  id SERIAL PRIMARY KEY,
  username TEXT NOT NULL,
  user_id INTEGER NOT NULL,
  first_login TEXT NOT NULL,
  last_login TEXT NOT NULL,
  count INTEGER DEFAULT 1
);

-- 留言板
CREATE TABLE IF NOT EXISTS guestbook (
  id SERIAL PRIMARY KEY,
  username TEXT NOT NULL,
  user_id INTEGER NOT NULL,
  body TEXT NOT NULL,
  time TEXT NOT NULL
);

-- 文章评论
CREATE TABLE IF NOT EXISTS comments (
  id SERIAL PRIMARY KEY,
  post_id INTEGER NOT NULL,
  username TEXT NOT NULL,
  user_id INTEGER NOT NULL,
  body TEXT NOT NULL,
  time TEXT NOT NULL
);

-- 用户表
CREATE TABLE IF NOT EXISTS users (
  id SERIAL PRIMARY KEY,
  username TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  role TEXT DEFAULT 'user',
  created_at TEXT NOT NULL
);

-- 文章表
CREATE TABLE IF NOT EXISTS user_posts (
  id INTEGER PRIMARY KEY,
  title TEXT NOT NULL,
  excerpt TEXT NOT NULL,
  tags TEXT NOT NULL,
  date TEXT NOT NULL,
  readTime TEXT NOT NULL,
  emoji TEXT NOT NULL,
  body TEXT NOT NULL
);

-- 开放公开读写权限（静态博客场景）
ALTER TABLE visits ENABLE ROW LEVEL SECURITY;
ALTER TABLE guestbook ENABLE ROW LEVEL SECURITY;
ALTER TABLE comments ENABLE ROW LEVEL SECURITY;
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE user_posts ENABLE ROW LEVEL SECURITY;

CREATE POLICY "public_select" ON visits FOR SELECT USING (true);
CREATE POLICY "public_insert" ON visits FOR INSERT WITH CHECK (true);
CREATE POLICY "public_delete" ON visits FOR DELETE USING (true);
-- ...（其余表同理）
```

## 七、注意事项

1. **GitHub Pages 构建**：推送后需等待 1-2 分钟才能看到更新
2. **云同步延迟**：数据写入是异步的，跨设备同步可能有数秒延迟
3. **RLS 策略**：当前为公开读写（适合静态博客），生产环境建议收紧
4. **密码存储**：用户密码使用简单哈希 `simpleHash()`，后续可升级为 bcrypt
5. **Supabase 免费限制**：500MB 数据库、5GB 带宽/月，个人博客完全够用
